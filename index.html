Enterprise Data Lakehouse & Reconciliation

Problem

A financial client's legacy batch processes (DB2/SQL Server) were failing to process high-volume daily transaction data, causing severe delays in reconciliation and compliance reporting.

Approach

Architected a Medallion architecture (Bronze/Silver/Gold) on Azure — Azure Data Factory to ingest on-premise data into ADLS, and PySpark jobs on Databricks handling deduplication, SCD Type 2, and aggregations into Delta tables.

Result

Modernized the analytics infrastructure, substantially cutting daily reconciliation processing time and providing one performant source of truth for Power BI reporting.

Streaming Pipelines & Cloud Data Warehouse

Problem

Fraudulent claims were only caught through nightly batch jobs, and legacy on-premise reporting databases were becoming too slow and expensive to scale.

Approach

Migrated workloads to AWS — a Kafka event pipeline (on EC2) streaming claim events into S3, with ETL normalizing fact/dimension tables into Redshift, tuning distribution/sort keys to match query patterns.

Result

Fraud-detection lag cut from a full 24-hour batch cycle to near real time, with noticeably faster analytical queries.
